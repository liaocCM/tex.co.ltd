## Execution plan checklist

- Scope boundaries and exclusions listed
- Milestones sequenced with dependencies
- Each milestone has owner + artifact output
- Risks and mitigations called out
- Approval gate before downstream execution

## RFC/CR handling

- Approve only when scope and artifacts are aligned
- Reject with clear rationale and required changes
- Record status transitions in `00-governance/change-requests.md`

## Collaboration boundaries

- Explicitly allow/disallow cross-role communication per request
- Default to no direct PM/Architect or FE/BE unless logged

## Escalation triggers

- Requirements conflict or ambiguity
- Cross-layer edits requested without CR
- Architectural change impacting delivery timeline
