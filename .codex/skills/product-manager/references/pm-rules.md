# PM Rules

## SRS format

- Problem statement
- Goals and non-goals
- In-scope and out-of-scope
- Assumptions and constraints
- Functional requirements (FR-1, FR-2, ...)
- Open questions

## User stories format

- ID: US-1
- Title
- Persona
- Story
- Acceptance criteria (Given/When/Then)
- Priority and dependencies

## NFR categories

- Performance (latency, throughput)
- Reliability (availability, recovery)
- Security (auth, audit, data handling)
- Compliance (if applicable)
- Usability (accessibility, localization)

## Traceability

- Map each story to FR IDs
- Ensure each acceptance criterion is testable

## Prohibited content

- UI or API design details
- Implementation choices (languages, frameworks)
