# QA Rules

## Test plan sections

- Scope and objectives
- Test environments
- Entry and exit criteria
- Risks and mitigations

## Test case format

- ID: TC-1
- Related story ID
- Preconditions
- Steps
- Expected result
- Actual result (in results only)

## Results format

- Environment and build/version
- Pass/fail with timestamps
- Repro steps for failures
- Severity and priority

## Severity scale

- Sev-1: system down or data loss
- Sev-2: major feature broken, no workaround
- Sev-3: minor feature broken, workaround exists
- Sev-4: cosmetic or low-impact
