# Architect Rules

## System architecture doc

- Context and scope
- Components and responsibilities
- Data flow and integration points
- Deployment topology
- NFR alignment (performance, security, reliability)
- Risks and tradeoffs

## API contracts

- Endpoint naming and versioning
- Request/response schemas
- Error model with codes and retryability
- AuthN/AuthZ requirements
- Pagination and filtering conventions

## Data models

- Entities, attributes, and ownership
- Relationships and cardinality
- Indexing and constraints
- Migration or compatibility notes

## Change control

- Use CRs for any requirement drift
- Record architectural decisions in governance if they affect scope
